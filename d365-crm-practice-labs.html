Every lab needs a real Power Platform environment: a free Developer Plan or trial works fine — nothing here needs production access, and several labs suggest a second environment as optional rather than required.

Lab F5

Your first client script

Warn the user in-form when a reminder is overdue.

 ~20 min ✓ Done ›
Scenario: Warn the user the moment they open a reminder whose date is in the past, using a namespaced JavaScript web resource — a good first practice for Foundations Module 6's script-vs-plugin distinction.

Prerequisites

Lab F2 completed.

Exercise 1 · Write the script
JavaScript · web resource
var Contoso = window.Contoso || {};
(function () {
    this.RenewalReminder = this.RenewalReminder || {};
    this.RenewalReminder.formEvents = {
        onLoad: function (executionContext) {
            var formContext = executionContext.getFormContext();
            var dateAttr = formContext.getAttribute("oh_reminderdate");
            var statusAttr = formContext.getAttribute("oh_status");
            if (!dateAttr || !statusAttr) return;

            var reminderDate = dateAttr.getValue();
            var isPending = statusAttr.getValue() === 100000000; // Pending
            if (reminderDate && isPending && reminderDate < new Date()) {
                formContext.ui.setFormNotification(
                    "This reminder's date has already passed.",
                    "WARNING",
                    "overdueReminder"
                );
            }
        }
    };
}).call(Contoso);
Exercise 2 · Upload as a web resource
Exercise 3 · Wire it to the form
Exercise 4 · Test
Tip: this is exactly the kind of rule that belongs client-side, not in a plugin — it's UX polish, not something that must hold true no matter how the record changed. See Foundations Module 6 for that distinction.
Success check: opening an overdue, still-Pending reminder shows a yellow form notification; a non-overdue one doesn't.

var request = {

Lab A5

Query data with the Web API and FetchXML

Pull the same overdue-reminder count two different ways.

 ~15 min ✓ Done ›
Scenario: Contoso's ops team wants a quick count of overdue reminders without opening the app. Get comfortable pulling data both ways: OData and FetchXML.

Prerequisites

Foundations Lab F2 completed. Your environment's Instance URL from Lab F1.

Exercise 1 · Web API in the browser console
JavaScript · browser dev tools console, from any Dataverse page
fetch("/api/data/v9.2/oh_renewalreminders?$select=oh_name&$filter=oh_status eq 100000003&$count=true", {
    headers: { "OData-MaxVersion": "4.0", "OData-Version": "4.0", "Accept": "application/json" }
}).then(r => r.json()).then(console.log);
Exercise 2 · The same query as FetchXML
FetchXML
<fetch aggregate="true">
  <entity name="oh_renewalreminder">
    <attribute name="oh_renewalreminderid" alias="count" aggregate="count" />
    <filter>
      <condition attribute="oh_status" operator="eq" value="100000003" />
    </filter>
  </entity>
</fetch>
Exercise 3 · Compare
Success check: both queries return the same overdue-reminder count.
